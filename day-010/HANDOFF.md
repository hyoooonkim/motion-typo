## v5에서 바뀐 것 (렌더링 엔진)

### 왜 바꿨나
v4는 "텍스트에 색을 입히는 방식"이었는데, 레퍼런스(열화상 손/몸)는 "텍스트 자체가 열을 발하는 물체"여야 함. Canvas 2D로 흉내내면 결국 "열화상 스타일 일러스트"지 "열화상 그 자체"가 안 나와.

v5에선 **텍스트를 셰이더 안에 넣어서 진짜 열원으로 처리**.

### 구조 전환

| | v4 | v5 |
|---|---|---|
| BG 열맵 | Canvas 2D 그라디언트 | **WebGL 셰이더 (thermalRamp)** |
| 텍스트 4레이어 | Canvas 2D shadowBlur 다중 패스 | **셰이더 내 multi-radius 텍스처 샘플링** |
| 텍스트 색상 | 온도별 PALETTE에서 선택 | **단일 heat field가 thermalRamp로 자동 매핑** |
| 컬러 보간 | CPU에서 단계별 | **GPU에서 픽셀별 연속 보간** |

### 핵심 아키텍처

```
[숨김 캔버스에 흰색 텍스트 렌더]
      ↓
[GPU 텍스처로 업로드 (LUMINANCE)]
      ↓
[프래그먼트 셰이더에서 multi-radius 샘플링]
   - core (r=0.4) : 텍스트 본체 = 최고온
   - mid  (r=1.5) : 가까운 헤일로
   - far  (r=4.0) : 중간 헤일로
   - wide (r=9.0) : 먼 번짐
      ↓
[ambient + textHalo + convection + fingerHeat 합성]
      ↓
[thermalRamp(heat) → 단일 컬러램프]
```

화면의 모든 픽셀이 **하나의 통합된 heat field**에서 나옴. 레퍼런스의 손이 작동하는 원리와 동일.

### 다이어트 효과 부수 효과
weight 변할 때마다 텍스트 텍스처 재생성 → GPU 재업로드 → 셰이더가 읽는 열원 면적이 자동 축소 → **다이어트하면 화면이 살짝 식는 시각적 인과관계** 발생. 컨셉(살이 빠진다)과 정확히 맞아떨어짐.

## v6에서 바뀐 것 (UI / 분위기)

### 1. 우측 큰 세로 온도계

작은 가로 막대 → **화면 우측 중앙에 큰 세로 온도계**로 교체. 키로 v4에 있던 우측 온도계 디자인이 시각적으로 강력해서 정식 메인 게이지로 승격.

```
[검은 트랙 18px × 50vh] ─ 71 °C
  - 컬러 스펙트럼이 clip-path로     HOT
    아래에서 위로 차오름
  - 30/40/.../110°C 틱 마크
  - 30/50/70/90/100/110만 라벨 노출
  - 흰 삼각형 인디케이터가 현재
    온도 가리킴
  - 상태 라벨: COLD/COOL/WARM/
    HOT/VERY HOT/CRITICAL
```

큰 숫자: `clamp(60px, 7vw, 110px)` Share Tech Mono. 뷰포트 크기에 따라 자동 스케일.

### 2. 셰이더 내 수증기 레이어

별도 SVG 오버레이 대신 **단일 셰이더 안에 통합**. 이유:
- 수증기가 thermal heat field와 같은 공간에 존재 → 텍스트와 함께 호흡
- GPU에서 매 프레임 새로 계산 → 정말로 흘러 다님

구현:
```glsl
// 3옥타브 simplex noise — 부풀어 오르는 구름
fogUv.y -= u_time * 0.045;  // 위로 drift
fogUv.x += sin(uv.y * 3.0 + u_time * 0.1) * 0.04;  // sway
float fog = snoise(fogUv * 1.4) * snoise(fogUv * 3.5) * snoise(fogUv * 7.0);
fog = pow(fog, 1.6);

// 상부 집중 (hot air rises)
fog *= 0.4 + topBias * 0.8;

// 온도 곡선: 60°C 이하 거의 없음, 80°C부터 본격적, overheat에서 max
float fogAmount = smoothstep(0.25, 0.95, u_temp) * 0.55 + u_overheat * 0.4;

// 색: 차가울 땐 푸른 흰색, 뜨거울 땐 따뜻한 흰색
vec3 fogColor = mix(vec3(0.85, 0.92, 1.0), vec3(1.0, 0.95, 0.88), u_temp);
col = mix(col, fogColor, clamp(fog, 0.0, 0.85));
```

### 3. 라이브 모니터 (좌하단)

작은 보더 박스 메타태그(HR/KCAL/MIN) → **각 항목 독립 행, 28px 큰 숫자**:

```
VITALS / SESSION
─────────────
HR    142  BPM     ← 심박 펄스 애니메이션
KCAL   23  CAL     ← 1칼로리마다 깜빡 flash
TIME 02:14 MIN
PASSIVE EXERCISE PROTOCOL
```

**온도 따라 색 변화** (3 tier):

| 온도 | tier | stat 색상 |
|---|---|---|
| <60°C | — | 흰색 |
| 60–80°C | warm | 옐로우 + 글로우 |
| 80–100°C | hot | 오렌지 + 글로우 |
| 100°C+ | crit | 핑크 + 빨강 외곽 글로우 |

세 stat이 동시에 같은 색으로 달궈짐 → "내 몸의 데이터가 같은 온도로 가열되고 있다"는 시각적 통일.

**살아있는 마이크로 인터랙션:**
- **HR 펄스 속도 = 60/bpm**: JS에서 `animationDuration = (60/bpm).toFixed(2)+'s'`. 72bpm이면 0.83초, 145bpm이면 0.41초마다 박동. 마우스 올릴수록 심장 빨라지는 게 보임.
- **KCAL 틱 flash**: 1칼로리 증가 감지 시 reflow 트리거 후 0.15s flash 애니메이션 재시작.

### 4. 텍스트 좌측 시프트
우측 온도계가 시각적 무게를 차지하니까 메인 텍스트를 좌측으로 90px 시프트:
```js
const cx = w / 2 - Math.min(w * 0.06, 90);
```
"텍스트 좌측 / 온도계 우측"의 비대칭 구도.

**1. Ping-pong FBO로 진짜 열 시뮬레이션**

- 작품을 한 단계 끌어올리는 가장 큰 한 방

**2. 사운드 — 저주파 + 멀리서 물 끓는 소리**
- 지글지글은 후라이팬 느낌. 사우나는 **저주파 hum + 가끔 들리는 물방울**이 정확
- Tone.js로 onset 트리거: 마우스 Y 위로 갈수록 hum 볼륨 + LFO 속도 증가
- 100°C 초과 시 고주파 ringing 추가
- HR 펄스와 동기화된 저음 thump도 좋음

**3. 추가 텍스트 페이드인 (기사 본문 발췌)**
- 메인 카피 외에 기사 다른 문장들 (예: "30분 = 운동 효과", "심박수 상승")
- 60°C 넘을 때부터 화면 모서리에 작게 fade-in
- 운동 데이터 메타포 강화

### 신중히 검토할 것

**4. Overexpose 5초 후 흰색 (v4 다음 확장)**
- 멋있긴 한데 컨셉 강화 못 함. "기절"이면 OK, "살이 빠져 사라짐"이면 다이어트 효과랑 충돌
- 대안: 100°C 초과 상태에서 **온도계만 105, 110, 115°C... 계속 올라가는데 화면은 더 이상 변하지 않음**. 정적인 공포가 컨셉에 더 맞을 수도

### 마이너 폴리싱

5. **수직 대류 강화** — 셰이더 `uv.y * u_temp * 0.22` 계수를 0.3까지 올리면 천장이 뚜렷하게 더 뜨거워짐 (사우나 사실성)
6. **모바일 터치 반응성** — 현재 lerp 계수 0.08, 모바일에서 0.12로 높이면 더 즉각적
7. **dpr 캡 조정** — 고해상도 디스플레이에서 fps 떨어지면 `Math.min(dpr, 1.5)`
8. **수증기 안에 물방울 흐림** — `dFdx/dFdy`로 수증기 짙은 영역만 살짝 distort

## 알려진 한계

- WebGL1 사용 (호환성 우선). WebGL2로 가면 더 큰 blur kernel 가능
- 텍스트 텍스처 재생성 cost: 다이어트 진행 중엔 매 ~4 weight마다 GPU 업로드. 60fps 유지되지만 저사양 모바일에선 throttle 필요할 수 있음
- `letterSpacing` 캔버스 API는 일부 구형 브라우저 미지원 → fallback 처리됨
- 셰이더 내 수증기 노이즈가 매 프레임 3옥타브 → 저사양 GPU에서 50fps 정도

## v7에서 바뀐 것 (ping-pong 잔열)

### 구조
```
PASS 1 (sim, 256×144 FBO)
  prev heat → 9-tap 확산 → 감쇠(0.985/60fps) → 텍스트·마우스 주입 → next heat
PASS 2 (display, 화면 해상도)
  heat field + text core + convection + steam → thermalRamp → 최종
```

### 얻은 것
- **텍스트 halo가 물리적**: 멀티 반경 샘플링 4회 → FBO 확산 1회로 교체. 열 번짐이 수학적으로 일관됨
- **잔열 트레일**: 마우스 궤적이 화면에 남음. 상단으로 갈수록 강하게, 감쇠로 천천히 식음
- **상승 대류 모방**: 확산 시 상단 샘플 비중 +8%
- **dt 기반 감쇠**: 120Hz/60Hz 독립

### 파라미터
- HEAT_W = 256 (4:3~16:9 화면비에 따라 H 자동)
- decay = pow(0.985, dt*60)
- mouseHeat = 0.12 + tempNorm*0.22 + overheat*0.3
- finger 반경: exp(-md*18) (aspect 보정됨)
