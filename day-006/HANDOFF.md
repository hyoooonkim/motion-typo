# 프로젝트 인수인계 — 폰이랑 이혼하고 싶습니다

## 작품 개요

**제목**: DIVORCE FROM PHONE — 협의이혼의사확인신청서 (인터랙티브 소장)
**원본 소스**: 어거스트 뉴스레터 "폰이랑 이혼하고 싶습니다" (https://august.stibee.com/p/373/)
**컨셉**: 스마트폰 중독을 다룬 기사를, 평범한 사람의 이혼소장처럼 시작했다가 피고가 스마트폰임이 드러나는 인터랙티브 타이포그래피 작품으로 재해석.

---

## 핵심 컨셉

1. **블랙 배경 / 화이트 A4 종이** — 법정 책상 위의 서류 같은 무드
2. **타자기 톤** — 모노스페이스 폰트, Web Audio API로 합성한 타자기 소리
3. **클릭 기반 진행** — 스크롤이 아니라 화면 클릭할 때마다 다음 줄이 타이핑됨
4. **한컴 조판부호** — 파란색 → ↵ ˙ (탭/엔터/공백), 주황색 [쪽 1] [수정] 표시로 정부 공문서 느낌
5. **5장의 종이가 겹쳐 쌓이는 구조** — 책상 위에 종이가 한 장씩 놓이는 물리적 감각
6. **검정 redaction + 빨간 재작성** — 사람 이혼으로 작성하다가 검정 박스로 지우고 빨간 글씨로 진실(핸드폰과의 이혼) 덧붙이기
7. **SVG 증거자료** — 스크린타임, 알림, MRI, 영수증 등 모두 inline SVG로 그려짐 (외부 이미지 의존 없음)
8. **3D 갤러리 피날레** — 마지막에 모든 종이와 증거가 3D 공간에 떠 있고 트랙패드로 시점 회전, 클릭으로 자료 열람

## 진행 흐름 (5페이지)

### 페이지 1 — 당사자
- 사건명, 원고(김민수, 마포구 합정동 주소) 검정 타이핑
- "피고 이___" 까지 치다 멈춤
- 낮은 베이스음 + 우상단에 [수정] 라벨 등장
- 검정 박스로 가리고 → **"Apple iPhone 15 Pro"** 빨간 재작성
- 주소도 "화면 앞 30cm, 베개 옆, 변기 옆"으로 정정
- 사건본인: ①시간 ②수면 ③집중력 ④허리디스크 ⑤시력

### 페이지 2 — 청구취지
- 처음엔 평범한 이혼 청구 (위자료, 친권, 양육비 등)
- 폭로 → 검정 박스 → 빨간 재작성
  - 시간 14,892시간 반환 / 숏폼 차단 / 23시 접근금지 / 비용 부담

### 페이지 3 — 청구원인 (좌측 본문 + 우측 증거 3장)
- 사람 이혼처럼 시작 → 폭로
- 빨간 청구원인 1~3번 + 우측에 폴라로이드 증거 사진 등장:
  - 甲第1호증: 스크린타임 스크린샷
  - 甲第2호증: 알림 발송 내역
  - 甲第3호증: 새벽 사용 기록

### 페이지 4 — 청구원인 계속 + 입증방법 (좌측 본문 + 우측 증거 2장)
- 청구원인 4~5번 (거북목, 인지능력)
- [입증방법] 헤더 + 甲第1~5호증 목록
- 우측에 증거 사진:
  - 甲第4호증: 경추 MRI
  - 甲第5호증: 디지털 디톡스 영수증

### 페이지 5 — 서명
- 2026년 4월 22일
- 원고 김민수 ⊙ (빨간 도장)
- 서울가정법원 귀중 (가운데 정렬)

### 피날레 — 3D 갤러리
- 휘잉~ 사운드와 함께 전환
- 어두운 공간에 5장의 종이 + 13개의 증거자료가 3D 공간에 산개
- 상단에 빨간 글씨 "이혼 인용"
- 트랙패드 두 손가락 스와이프 / 마우스 드래그로 시점 회전
- 카드 클릭 → 큰 화면으로 자료 열람

## 기술 구조

### 단일 파일
모든 게 `divorce.html` 한 파일에 들어있음. 의존성 없음.
- 구글 폰트만 외부에서 로드 (Nanum Myeongjo, Noto Serif KR, IBM Plex Mono)
- 모든 증거 이미지는 inline SVG data URL

### 주요 JS 구조
- **AudioEngine 클래스** — Web Audio API로 타자기/법봉/베이스음 합성
- **actions 배열** — 각 클릭마다 실행되는 async 함수들의 순서
- **typeInto(el, html, speedMs)** — 한 글자씩 타이핑 애니메이션
- **redactLine(el)** — 기존 라인을 검정 박스로 덮음
- **rewriteRed(el, html)** — 덮인 라인 아래에 빨간 글씨로 새로 작성
- **placeEvidenceOnPaper({...})** — 페이지에 폴라로이드 증거 사진 놓기
- **newPage(n)** — 새 종이를 겹쳐 쌓기
- **openGallery()** — 3D 갤러리 피날레 전환

## 개발 히스토리 요약

초기 컨셉 5개 중 "이혼 서류 양식 + 무한 스크롤 트랩" 선택 → 공문서 디자인으로 여러 차례 이터레이션 → 기획 전면 재설계 → 타자기 + 블랙박스 redaction + 빨간 재작성 + 3D 갤러리 피날레로 정착.

주요 수정 포인트:
- 글리치 효과 제거, 공문서 톤으로 정리
- 종이 화이트로 변경
- 여백 강조, 불필요한 라인 최소화
- 인트로 화면 제거 → 바로 1페이지부터 시작
- 페이지 겹침 방식 (옆으로 밀리는 게 아니라 책상 위에 쌓이듯)
- redact 시 페이지 흔들림 제거
- Unsplash 외부 이미지 → 모두 SVG로 직접 제작 (403 이슈 해결)
- 4페이지 overflow 문제 → 5페이지로 분리

## 다음 AI에게 전달 시 참고사항

1. **코드 수정 시 액션 순서 주의** — `actions` 배열 안 async 함수들은 순서대로 실행되므로 위치가 중요함
2. **redact는 "기존 텍스트 위에 검정 박스"**, **빨간 재작성은 "그 아래에 새 빨간 라인"** 구조. 동일 라인 위에 덮어쓰는 게 아님
3. **3, 4페이지의 본문 라인 너비 제한**은 `.page.has-evidence .line:not(.section-head):not(.full-width)` 셀렉터에 있음
4. **페이지 간 여백이나 위치 조정**은 `pageOffsets` 배열에서
5. **사운드 이슈가 생기면** — AudioContext `resume()` 호출 여부, throttle 시간 확인
