# Day 004 — 귀여운 얼굴에 만지는 재미는 덤

## 작품 개요
- **텍스트**: "귀여운 얼굴에 만지는 재미는 덤"
- **컨셉**: 텐저블 타이포그래피 · 13가지 재질 · 얼굴↔텍스트 전환
- **인터랙션**: 호버 (재질별 모션) · 클릭 (레이아웃 전환) · 사운드

## 핵심 메커니즘
13글자 각각에 다른 3D 재질 이미지(PNG) 매핑. 초기에는 글자들이 **얼굴 형태**로 모여 있다가, 클릭하면 3줄 텍스트 레이아웃으로 전환.

### 얼굴 레이아웃
- 귀여운 → 왼쪽 눈
- 얼굴에 → 오른쪽 눈
- 만지는 → 코 (일자 배열)
- 재미는 덤 → 입 (웃는 모양)

### 텍스트 레이아웃
- 1줄: 귀여운 얼굴에 (6글자)
- 2줄: 만지는 재미는 (6글자)
- 3줄: 덤 (가운데 정렬, 1글자)

## 기술 스택
- CSS transform 기반 (Three.js / Canvas 그리드 메시는 시도 후 폐기)
- `position: absolute`로 좌표 기반 배치
- Web Audio API로 효과음 합성
- 투명 PNG 사용 (`mix-blend-mode: normal` — screen 모드는 겹침 시 투명 문제 발생)

## 에셋
- `01_fur.png` ~ `13_cotton.png` — AI 생성 투명 PNG
- `letter.svg`, `letter.ai` — 글자 위치 참고용
- `source/` — 원본 이미지
- `test/` — 테스트 버전

## 주요 이슈/러닝
- **투명 처리**: `mix-blend-mode: screen`으로 블랙 배경 제거 시 글자 겹침에서 투명해지는 문제 → 배경 제거한 투명 PNG로 근본 해결
- **레이아웃 실수**: 초기에 5줄로 배치한 버그 → SVG 원본 참고해서 3줄로 교정
- **Three.js / Canvas 메시** 시도: 퍼포먼스 이슈와 구현 복잡도로 폐기
- **사운드 활성화**: 첫 클릭으로 `audioCtx.resume()` 필수
- **Web Audio 차단**: 사용자 제스처 없이는 오디오 재생 불가
